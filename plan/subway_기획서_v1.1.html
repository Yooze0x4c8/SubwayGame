01

개요 · 확정 사항

v0.4의 결정 8건을 닫고 와이어프레임을 반영한 뒤, v1.1에서 시계 구조를 개정했습니다.

한 줄 정의

서버가 지정한 역·호선에서 출발해 순서대로 이어지는 역 이름을 대고, 못 대는 사람이 나오는 즉시 라운드가 끝나는 실시간 대전 게임입니다. 5라운드 누적 점수 1위가 승리합니다.

02

게임 구조

서든데스가 이 게임의 전부입니다. 나머지 규칙은 전부 여기서 파생됩니다.

게임  = 고정 R라운드 (방장: 3 / 5 / 7, 기본 5)
라운드 = 시작 역·호선 지급 → 라운드 시계 120초 시작 → 턴 반복
        ├─ 누군가 실패하는 순간 종료  (서든데스 · −D + 끝내기 보너스)
        └─ 라운드 시계 소진       (완주 종료 · 실패자 없음, 차감 없음)
승자  = R라운드 누적 점수 1위
  • 라운드마다 used 초기화, 시작 역·호선 재추첨
  • 선공 로테이션 필수 — 라운드가 5턴에 끝나면 뒤 3명은 턴도 못 받습니다. 없으면 뒷자리가 구조적으로 손해입니다
  • 마지막 라운드 전 점수 ×1.5 — 역전 장치

실패 정의

턴 시간 초과만 실패입니다. 오답 입력(미등재·중복·노선 불일치)은 시간만 소모하고 재입력할 수 있습니다. 시도 자체는 무제한입니다.

라운드 시계 만료는 실패가 아닙니다. 아무도 못 대서 끝난 게 아니라 시간이 다 된 것뿐이므로, 차감도 끝내기 보너스도 없습니다. 각자 그 라운드에 번 점수를 그대로 들고 다음 라운드로 갑니다.

데드락 로직이 존재하지 않습니다. 아무도 답할 수 없는 상태 = 현재 차례가 시간 초과 = 라운드 종료. 서든데스가 데드락 처리를 통째로 흡수합니다. v0.3의 "전원 1바퀴 연속 패스" 카운터는 0줄입니다.
03

타이머

◎ 환승역 — 시계를 라운드와 턴으로 나눈 것이 v1.1의 핵심 변경입니다.

두 개의 시계

라운드 시계  R₀ = 120초                          // 라운드 전체 예산. 방장: 90 / 120 / 180
턴 시계    T(n) = max(5, round(T₀ × r^n))      // 한 사람이 한 번 답하는 시간. T₀ = 15초
기호의미값
R₀라운드 시계 · 방장 설정90 / 120 / 180초
T₀턴 시계 초기값 · 방장 설정10~30, 기본 15초
n라운드 내 누적 턴 인덱스 (0부터)—
r턴당 감쇠0.92 / 0.96 / 0.98
T_min턴 시계 하한5초
  • 라운드 시계는 개시 게이트입니다. "새 턴을 시작할 수 있는가"만 판정합니다. 턴 도중에 0이 되어도 그 턴은 온전한 T(n)을 받고 끝까지 진행합니다.
  • 턴을 넘길 때 라운드 잔여가 0 이하면 → 완주 종료. 실패자 없음, 차감 없음, 끝내기 보너스 없음.
  • 정산·연출 중에는 라운드 시계가 멈춥니다. 라운드 실제 최대 길이는 R₀ + T_max ≈ 135초.
개시 게이트로 잡은 이유. 라운드 시계가 턴 도중 만료되면 그 턴을 실패 처리한다고 하면, 2초짜리 턴을 받은 사람이 −D를 맞습니다. 반대로 "만료 즉시 무승부 종료"로 하면 마지막 턴에 일부러 시간을 끌어 실패를 회피하는 구멍이 생깁니다. 게이트 방식은 두 문제를 동시에 없앱니다 — 모든 턴은 항상 온전한 턴 시계를 받고, 시간을 끌어도 자기 턴은 자기가 답해야 합니다.

왜 120초를 턴이 아니라 라운드에 주는가

v1.0은 턴 하나에 최대 120초를 줬습니다. 두 가지가 깨집니다.

  • 버티기가 가능해집니다. 답을 모르면 아예 입력을 안 하고 120초를 끝까지 태우는 게 이득인 상황이 생깁니다 — 어차피 실패할 거면 남들 시간을 뺏는 편이 낫고, 선두라면 라운드를 늘어뜨려 판을 식히는 데 쓸 수도 있습니다.
  • 초반이 통째로 대기 시간입니다. 8인방 첫 바퀴에만 최악 16분입니다. 답을 아는 사람도 자기 차례를 16분 기다립니다. 실시간 게임에서 가장 비싼 실수입니다.

그리고 긴 턴 시간은 난이도를 만들지 않습니다. 난이도를 만드는 건 역 소진입니다(아래). 역이 널널한 라운드 초반에 생각 시간 15초와 120초는 아무 차이도 만들지 않습니다 — 그냥 늘어질 뿐입니다.

대신 게임 길이가 확정됩니다. 120초 × 5라운드 + 정산 ≈ 12분. v1.0의 최악값 35분(1라운드)과 비교하면 성격이 다른 제품이 됩니다. 지인방이 "한 판 더" 하기 딱 좋은 길이이고, 방장이 방 만들 때 몇 분짜리인지 말할 수 있게 됩니다.

왜 감쇠 단위는 바퀴가 아니라 턴인가

이 게임의 난이도를 만드는 건 역 소진이고, 소진량은 누적 턴 수입니다. 인원과 무관합니다. 바퀴 단위로 감쇠하면 두 압박이 어긋납니다.

5바퀴 시점누적 턴소진된 역턴 시계
8인 · 바퀴 단위4040역7초
2인 · 바퀴 단위1010역7초

2인방은 역이 10개밖에 안 줄었는데 시계는 7초입니다. 시간 압박만 있고 소진 압박이 없습니다. 턴 단위로 하면 40턴째에 인원과 무관하게 40역이 사라졌고 시계도 똑같이 짧습니다.

라운드 시계 120초 안에서 한 라운드는 대략 15~30턴입니다. 곡선의 유효 구간은 그 범위이고, 40턴 이후는 라운드 시계가 먼저 끝나서 거의 도달하지 않습니다.

라운드 간 감쇠 a는 폐기합니다 v1.1

v1.0에서 a=0.75를 확정한 근거는 "매 라운드 초반 120초 늘어짐이 5번 반복된다"였습니다. 라운드 시계가 그 문제를 직접 없앴으므로 a는 목적을 잃었습니다.

게다가 T₀=15에 a=0.75를 걸면 라운드 시작 턴 시계가 15 → 11 → 8 → 6 → 5가 되어 4~5라운드는 첫 턴부터 하한 5초로 시작합니다. 이건 고조가 아니라 파탄입니다. 게임 전체의 고조는 마지막 라운드 ×1.5가 담당하고, 방장이 더 조이고 싶으면 라운드 시계를 90초로 내리면 됩니다.

4.3 "직전 정답자 +20"이 왜 필수인가

차감만 있으면 고의 패스가 최적 전략이 되는 구멍이 생깁니다. 내가 꼴찌인데 1등이 점수를 벌고 있으면, 일부러 패스해서 라운드를 끝내면 −17 손해로 상대의 잔여 득점 수십 점을 차단할 수 있습니다.

이중 방어

  • 고정 R라운드가 1차입니다. 일찍 끝내도 다음 라운드가 바로 시작되므로 상대 득점 기회를 "차단"한 게 아니라 "리셋"한 것뿐입니다. 나만 −D 손해 → 순손해.
  • +20 / +5가 2차입니다. 8인이면 실패 순간 남들에게 총 +55가 뿌려집니다. 고의 패스 = 자살.
  • 라운드 잔여 비례 차감이 3차입니다 v1.1. 고의 패스가 가장 이득인 시점(라운드 초반, 상대의 잔여 득점 기회가 클 때)이 정확히 가장 비싼 시점(−48)입니다. 차감액이 "내가 없앤 기회의 크기"를 그대로 따라갑니다.

그리고 +20은 원래 의도한 전략을 정확히 보상합니다 — "다음 사람이 답하기 어려운 상태로 넘긴 사람"이 최대 수혜자가 됩니다. 소형 노선으로 몰아넣기, 환승역 선점, 긴 역 선점이 전부 여기로 수렴합니다.

// 유효: lines(T) ∩ lines(S) ≠ ∅  AND  T ∉ used  AND  T ∈ 등재역(권역)
Set<Line> X = A ∩ lines(T);
if (X != ∅) { A' = X;                    transfer = false; }  // 직진
else        { A' = lines(S) ∩ lines(T);  transfer = true;  }  // 🔁 환승
S' = T;
X = ∅인데 유효하다는 건 S가 활성 노선 외의 노선도 가졌다는 뜻 → S는 반드시 환승역입니다. "환승역에서만 갈아탄다"는 물리가 별도 조건 없이 자동으로 나옵니다.

서버 로직

서버 권위. 시간 초과 판정과 점수 계산은 전부 서버에서.

class GameState {
    long currentStationId, activeMask, usedLineMask;
    int  round, turnIndex, startPlayerIdx;   // 라운드마다 로테이션
    long roundDeadline;                      // 라운드 시계 (절대 시각)
    long turnDeadline, turnLimitMs;          // 턴 시계
}

// 턴 제한시간 — 라운드 간 감쇠 a 없음 (v1.1)
int turnLimit(int n) {
    return Math.max(5, Math.round(T0 * Math.pow(r, n)));
}

// 턴 개시 — 라운드 시계는 게이트일 뿐, 턴 시계를 자르지 않는다
void startTurn(GameState g, Player p) {
    if (now() >= g.roundDeadline) { finishRound(g); return; }   // 완주 종료
    g.turnLimitMs = turnLimit(g.turnIndex) * 1000L;
    g.turnDeadline = now() + g.turnLimitMs;                     // roundDeadline 초과 허용
}

// 턴 시간 초과 = 실패 (서든데스)
void onTurnTimeout(GameState g, Player p) {
    long remain = Math.max(0, g.roundDeadline - now());          // 라운드 잔여
    int  D   = clamp(Math.round(remain / 1000f * 0.4f), 10, 50); // v1.1: 기준이 라운드
    float mul = (g.round == lastRound) ? 1.5f : 1.0f;

    score(p,            -D * mul);
    score(lastAnswerer, +20 * mul);          // 끝내기 보너스
    others().forEach(o -> score(o, +5 * mul));

    endRound(g);                             // 데드락 처리 없음 — 이게 전부
}

// 라운드 시계 소진 = 완주 종료. 실패자도 차감도 없다
void finishRound(GameState g) { endRound(g); }
  • roundDeadline·turnDeadline 모두 절대 시각(epochMs). 클라이언트 타이머는 표시용일 뿐입니다.
  • 라운드 시계는 정산·연출 구간에서 정지합니다. 재개 시 roundDeadline += 정지시간.
  • 역 목록은 공개 데이터라 클라에 내려도 무방합니다. 판정만 서버에서 하면 됩니다.
  • 밸런스 상수 전부 설정 파일로 분리 — 15 / 20 / 10 / 차감계수 0.4 / R₀ / T₀ / r. 플레이테스트 전엔 아무도 모릅니다.
